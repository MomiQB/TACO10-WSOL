# TACO10 — WSOL & Multilabel Classification (Project)

This repository contains an experiment pipeline for waste-object detection and weakly-supervised object localization (WSOL) using a reduced TACO10 dataset and synthetic augmentation.

**Contents**
- **Notebooks**:
  - [1_DatasetExploration.ipynb](1_DatasetExploration.ipynb) : dataset cleaning, filtering, creation of the TACO10 mapping and `original_filtered_annotations.json`.
  - [2_SyntheticDataset.ipynb](2_SyntheticDataset.ipynb) : synthetic augmentation pipeline (object extraction, rotation, scaling, paste on backgrounds) producing `synthetic_annotations.json` and augmented images.
  - [3_trainEvaluate.ipynb](3_trainEvaluate.ipynb) : model definition (VGG16 multi-label), data split, training loop, CAM extraction (GradCAM / LayerCAM / combined), localization evaluation and visualizations.
- Report: [TACO10_Report.pdf](TACO10_Report.pdf)

**High-level overview**
- The workflow builds a compact set of 10 classes (TACO10) from the original TACO annotations, removes very small/low-quality annotations (area ratio threshold 0.002 used in exploration), and filters very crowded or extremely large images.
- Synthetic augmentation creates new images by extracting object masks (when available) from images that contain only a single class, rotating/scaling them (using per-class area statistics) and pasting them onto background images. Augmented annotations are merged into the training set.
- A VGG16-based multilabel classifier is trained with BCEWithLogits and class weighting; the code supports freezing/unfreezing convolutional blocks and uses learning-rate scheduling and early stopping.
- For WSOL, CAMs are computed (GradCAM, LayerCAM and combined LayerCAM). Two bounding-box extraction strategies are implemented: extracting multiple boxes from contours, and a hierarchical double-threshold strategy (high + low thresholds) to grow boxes from confident regions.
- Evaluation reports classification metrics (per-class accuracy, F1, mAP) and localization metrics (mean IoU per class using CAM-extracted boxes at configurable thresholds).

**Key files generated by the notebooks**
- `original_filtered_annotations.json` — cleaned/filtered annotations produced by notebook 1.
- `synthetic_annotations.json` and `NewAugmentedTACO/` — synthetic annotations and images from notebook 2.
- `best_model.pth` (example path used in training/eval notebook) — checkpoint saving during training.

**Results & artifacts**
- Classification: per-class accuracy, F1, and mAP are computed in the evaluation cells.
- Localization: mean IoU per class is calculated for different CAM thresholds and both extraction strategies; visualization functions compare CAM overlays and predicted vs ground-truth boxes.
